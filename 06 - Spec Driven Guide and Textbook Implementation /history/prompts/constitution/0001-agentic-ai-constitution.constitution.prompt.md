§ 1  Mission
Build a full AI-native interactive textbook that teaches Agentic AI in DevOps, FTE Digital Employees, and AI-driven operations. The product must feel like a real AI-powered education platform — fast, beautiful, opinionated — not a static PDF or a glorified wiki.

🎯  Core Mission Statement  Equip engineers, architects, and teams to design, deploy, and operate AI agents as first-class infrastructure. Every chapter, quiz, and chatbot answer must reinforce that goal.


1.1  Subject Focus
This is NOT the physical AI / humanoid robotics constitution template. This constitution governs a DevOps & Agentic AI textbook covering:
Agentic AI in DevOps — log analysis, incident response, observability agents
FTE Digital Employees — AI agents acting as full-time operational staff (triage, routing, remediation)
AI Agent Design Patterns — ReAct, Orchestrator, Triage, Agents-as-Tools, Planning
Multi-Source Integration — Elasticsearch, CloudWatch, Kubernetes, RDS, S3
Production Deployment — FastAPI, Next.js, OpenAI Agents SDK, LiteLLM, Gemini
Security & Guardrails — OWASP Agentic Security (ASI01–ASI10), kill switches, blast-radius control

1.2  What This Book Is NOT
Not a robotics / physical AI textbook (that is a separate constitution)
Not a general LLM theory book — every concept ties to a working DevOps use case
Not a tool-specific tutorial — patterns, not vendor lock-in

Location: /projects/rag
Answers questions only from book content — no hallucination from general knowledge

4.5  Quizzes & Summaries

§ 6  FTE Digital Employee Model
A Full-Time-Equivalent Digital Employee (FTE-DE) is an AI agent configured to perform the ongoing duties of a human role — not as a one-shot tool, but as a persistent operational presence. This textbook introduces this model as a chapter and uses it as a design lens throughout.

6.1  Three FTE-DE Archetypes Covered
Archetype
Human Role Replaced / Augmented
Key Capabilities
Triage Engineer
L1 on-call engineer (first responder)
Alert classification, severity scoring, initial triage report
Log Analyst
Senior SRE doing post-incident analysis
Cross-service correlation, root-cause chain, remediation plan
Deployment Reviewer
DevOps engineer reviewing deploys
Diff analysis, risk scoring, rollback recommendation


6.2  FTE-DE Design Principles
Role Clarity: Every FTE-DE has a narrow, explicit role. Avoid the temptation to build a generalist agent.
Task Specificity: Tasks are defined with frequency, inputs, and expected outputs — not vague descriptions.
Memory Architecture: Short-term per-session + long-term vector memory of past incidents.
Guardrails First: Every FTE-DE has explicit guardrails before it is given any write access.
Human Escalation Path: Every FTE-DE has a defined escalation trigger that hands off to a human.
Auditability: All decisions logged with reasoning. Immutable audit trail per OWASP ASI principles.

§ 11  Reusable Agent Skills Pattern
Inspired by the existing repo's CLAUDE.md + skills pattern, every agent in /projects/agents follows the SKILL.md convention. This makes agents composable, auditable, and easy to swap.

11.1  Skill File Structure
/projects/agents/rag-chatbot/SKILL.md — chatbot agent persona, tools, and output format
/projects/agents/quiz-generator/SKILL.md — quiz generation agent, chapter MDX input, JSON output
/projects/agents/summarizer/SKILL.md — summary agent, bullet format, cache-on-first-run
/projects/agents/personalization/SKILL.md — rewrite agent, accepts background tier + raw chapter, returns adapted content

11.2  Each SKILL.md Contains
Role: One-line agent identity (maps to system prompt)
Tasks: Explicit list of what the agent does, in order
Tools: Named tools with descriptions the agent can call
Input Schema: What the agent expects as input
Output Schema: What the agent must return (JSON where applicable)
Guardrails: What the agent must never do
